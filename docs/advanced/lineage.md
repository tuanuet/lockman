# Advanced: Lineage

Lineage is the advanced path for parent-child overlap rules.

Most users should stay on the default `github.com/tuanuet/lockman` path until they have a concrete need for parent-child lock semantics across resources or services.

At this stage, lineage remains a narrower advanced area than the default SDK onboarding path. The default docs and examples focus on `Run`, `Claim`, strict fenced execution, and composite sync execution first.
